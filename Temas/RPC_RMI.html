RPC Y RMI

Remote Produce Call & Remote Method Invocation

RPC

Es un programa que utiliza una computadora para ejecutar código en otra máquina remota sin tener que preocuparse por las comunicaciones entre ambas, el protocolo que se utiliza para esta llamada es un gran avance sobre los sockets de Internet usados hasta el momento.

COMUNICACIÓN ENTRE PROCESOS

    IPC: Inter-Process Communication
  • Procesos comparten datos
  • Cliente-Servidor (Petición-Respuesta)
    Multiples formas de compartir información
  • Archivos
  • Sockets
  • Tuberias
  • Colas de mensajes
  • Memoria Compartida
  • Paso de mensajes
    LLAMADA A PROCEDIMIENTO REMOTO
      Procesos comparten información como en IPC
    • Generalmente entre diferentes computadoras a través de una red

TIPOS DE SEMANTICA

    Semántica "tal-vez"

  • Procedimiento remoto puede ejecutarse una vez o ninguna vez.
  • El cliente puede recibir una respuesta o ninguna.
      • Funcionamiento
    1. El cliente envía una petición y se queda a la espera un tiempo determinado.
    2. Si no llega la respuesta dentro del tiempo de espera, continúa su ejecución.
    3. El cliente no tiene realimentación en caso de fallo (no sabe que pasó).

    Semántica "al-menos-una-vez

  • Procedimiento remoto se ejecuta una o más veces.
  • El cliente puede recibir una o más respuestas.
      • Funcionamiento
    1. El cliente envía una petición y queda a la espera un tiempo.
    2. Si no llega respuesta o ACK dentro del tiempo de espera, repite la petición.
    3. El servidor no filtra peticiones duplicadas (el procedimiento remoto puede ejecutarse repetidas veces).
    4. El cliente puede recibir varias respuestas.

    Semántica "como-máximo-una-vez"

  • El procedimiento remoto se ejecuta exactamente una vez o no llega a ejecutarse ninguna.
  • El cliente recibe una respuesta o una indicación de que no se ha ejecutado el procedimiento remoto.
      • Funcionamiento
    1. El cliente envía la petición y queda a la espera un tiempo.
    2. Si no llega respuesta o ACK dentro del tiempo de espera, repite la petición.
    3. El servidor filtra las peticiones duplicadas y guarda historial con las respuestas enviadas (servidor con memoria). El procedimiento remoto sólo se ejecuta una vez.
    4. El cliente sólo recibe una respuesta si la petición llegó y se ejecutó el procedimiento, si no recibe informe del error.

RMI

Es un mecanismo ofrecido por Java para invocar un método de manera remota que forma parte del entorno estándar de ejecución de Java y proporciona un mecanismo simple para la comunicación de servidores en aplicaciones distribuidas basadas exclusivamente en Java.

    Protocolo para aplicaciones distribuidas

  • Cliente-Servidor
  • El cliente invoca métodos en el servidor “como ” si fueran locales

    PARTICIPANTES

  • Servidor
  • Cliente
  • Rmiregistry
  • Interfaz remota

PROCEDIMIENTO

    Cliente

  • Conoctarse al registro
  • Obtener la referencia al servidor
  • Invocar los métodos requeridos

    Ejecución

  • Arrancar el registro
  • Arrancar el servidor
  • Arrancar el cliente

    Interfaz Remota

  • Hereda de java.rmi.Remote
  • Define los métodos ofrecidos por el servidor, y que podrán ser invocados por el cliente
  • Los métodos deben declararse con throws RemoteException

    Servidor

  • Implementa la interfaz remota
  • Crea el objeto remoto y lo exporta para que pueda recibir conexiones
  • Registra el objeto remoto en el registro para poder entregar su referencia a los clientes

    SEMANTICA DE RMI

  • Un cliente siempre hace referencia a un objeto remoto a través de una de las interfaces Remote que éste implementa.
  • Un método remoto siempre debe declararse indicando que puede lanzar una RemoteException.
  • Registra el objeto remoto en el registro para poder entregar su referencia a los clientes
  • Los clientes de objetos remotos deben capturar y tratar las excepciones remotas.
  • Un objeto pasado como parámetro a un objeto remoto o devuelto desde un método remoto debe ser serializable, o ser otro objeto Remote.
  • Un objeto pasado como parámetro a un objeto remoto o devuelto desde un método remoto lo hace valor en vez de por referencia, excepto si dicho objeto es remoto.
