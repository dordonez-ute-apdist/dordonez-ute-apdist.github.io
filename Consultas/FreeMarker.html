FREEMARKER

FreeMarker es un "motor de plantillas"; una herramienta genérica para generar salida de texto (desde HTML hasta código fuente generado automáticamente) basada en plantillas. Es un paquete de Java, una biblioteca de clases para programadores de Java. No es una aplicación para usuarios finales en sí misma, sino algo que los programadores pueden integrar en sus productos.

FreeMarker está diseñado para ser práctico para la generación de páginas web HTML , particularmente para aplicaciones basadas en servlets siguiendo el MVC.(Controlador de vista de modelo).

 

PROPÓSITO GENERAL

  •      Se puede utilizar para generar cualquier tipo de texto: HTML, XML, RTF, código fuente Java, etc.
  •      Fácil de integrar en su producto: ligero. No requiere entorno de servlet (no depende de clases javax.servlet).
  •      Cargador de plantillas conectable: puede cargar plantillas desde cualquier fuente; archivos locales, base de datos, etc.
  •      Puede hacer lo que quiera con el texto generado: almacenarlo en un archivo local, enviarlo como correo electrónico, enviarlo de vuelta a un navegador web desde una aplicación web, etc.








    • POTENTE LENGUAJE PLANTILLA

  •      Todas las directivas habituales: incluyen, if / elseif / else, construcciones de bucle.
  •      Creación y modificación de variables en plantillas.
  •      Puede usar expresiones complejas para especificar valores en casi todas partes.
  •      Operaciones de cadena: concatenación, subcadena, mayúsculas, mayúsculas, escape, etc.
  •      Cálculos aritméticos de precisión decimal.
  •      Aritmética booleana.
  •      Lectura de elementos de matriz asociativa y matriz.
  •      Matrices y matrices asociativas literales.
  •      Puede agregar sus propias funciones (métodos) para cálculos especiales.
  •      Macros con parámetros posicionales y con nombre y con contenido anidado (cuerpo).

    • POR EJEMPLO

    <@myMacro color = "red" width = 2> ...

  •     Espacios de nombres para ayudar a construir y mantener bibliotecas de macros reutilizables o para dividir grandes proyectos en módulos separados, sin preocuparse por conflictos de nombres.
  •     Bloques de transformación de salida. Realice transformaciones como escape HTML, compresión, resaltado de sintaxis, etc. en la salida generada por el fragmento de plantilla anidado. Puedes definir tus propias transformaciones.
