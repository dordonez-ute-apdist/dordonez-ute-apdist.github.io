DPC++

Que es DPC++


  • Data Parallel C ++ (DPC ++) es una alternativa abierta a los lenguajes propietarios de arquitectura única.
  • Permite a los desarrolladores reutilizar código en los objetivos de hardware (CPU y aceleradores como GPU y FPGA) y también realizar ajustes personalizados para un acelerador específico.
  • Incluye extensiones de lenguaje desarrolladas mediante un proceso de comunidad abierta. Diseñado a propósito como una alternativa abierta e intersectorial a los lenguajes patentados de arquitectura única, DPC ++ permite a los desarrolladores migrar más fácilmente su código a CPU, GPU y FPGA, y también ajustar el rendimiento para un acelerador específico.

    • El lenguaje se compone

  • C++

    • Cada programa DPC ++ es también un programa C ++. Una implementación compatible con DPC ++ debe admitir el lenguaje principal C ++ 17 (como se especifica en las secciones 1 a 19 de ISO / IEC 14882: 2017) o más reciente.

  • SYCL

    • DPC ++ se basa en la especificación SYCL de The Khronos Group. El lenguaje SYCL permite la definición de funciones paralelas de datos que se pueden descargar a los dispositivos y define las API y las clases en tiempo de ejecución que se utilizan para orquestar las funciones descargadas.

  • • Extensiones de lenguaje DPC ++.

    • Una implementación compatible con DPC ++ debe admitir las características de lenguaje especificadas. Estos incluyen memoria compartida unificada (USM), colas ordenadas y reducciones. Algunas extensiones son necesarias solo cuando la implementación de DPC ++ admite una clase específica de dispositivo.

    Viaje al paralelismo, la escalabilidad y la portabilidad

  • La programación paralela ha recorrido un largo camino durante varias décadas y ahora podemos evaluar mejor las formas más efectivas de respaldar la programación paralela de datos.
  • También debe tenerse en cuenta la naturaleza heterogénea de los sistemas informáticos actuales.
  • Ser eficaz tanto en el paralelismo de datos como en la programación heterogénea requiere buenas soluciones de escalabilidad y portabilidad.
  • El escalado es una medida de la eficacia de la programación paralela, que mide el grado en el que un programa utiliza con eficacia más hardware paralelo cuando está disponible.
  • Nunca debemos perder de vista el objetivo, para cualquier sistema de programación paralela, de permitir un escalado efectivo.
  • Esto incluye escalar a medida que movemos nuestra aplicación entre plataformas. Las plataformas heterogéneas confunden el problema porque diferentes arquitecturas pueden tener necesidades sustancialmente diferentes.
  • • No pretendemos que DPC ++ resuelva el escalado y todas las necesidades de plataformas heterogéneas para nosotros. Más bien, DPC ++ es una herramienta que nos permite tener éxito al ayudarnos a lidiar con problemas que de otra manera interferirían con nuestros objetivos de escalamiento y al ofrecernos técnicas para abordar las necesidades de plataformas heterogéneas.

    Cómo se relaciona OpenCL con la familia de estándares de aceleración de Khronos

    OpenCL proporciona a la industria la capa de ejecución ágil de procesadores 'cercana al metal' más baja para acelerar aplicaciones, bibliotecas y motores, y también proporciona un objetivo de generación de código para compiladores. A diferencia de las API 'solo para GPU', como Vulkan, OpenCL permite el uso de una amplia gama de aceleradores que incluyen CPU de múltiples núcleos, GPU, DSP, FPGA y hardware dedicado, como motores de inferencia.




    Flexibilidad de implementación

    A medida que el panorama de la industria de plataformas y dispositivos se vuelve más complejo, las herramientas están evolucionando para permitir que las aplicaciones OpenCL se implementen en plataformas que no tienen controladores OpenCL nativos disponibles. Por ejemplo, el compilador clspv de código abierto y el traductor de API clvk permiten que las aplicaciones OpenCL se ejecuten en un tiempo de ejecución Vulkan. Esto brinda a los desarrolladores de OpenCL una flexibilidad significativa sobre dónde y cómo pueden implementar sus aplicaciones OpenCL.



    Extensiones de lenguaje de kernel

    Algunas extensiones están disponibles para los estándares de lenguaje del kernel publicados existentes. Los compiladores y controladores compatibles pueden admitir opcionalmente las extensiones, por lo que existe un mecanismo para detectar su compatibilidad en el momento de la compilación. Los desarrolladores deben saber que es posible que no todas las extensiones sean compatibles con todos los dispositivos.
