OpenCL

¿INTRODUCCIÓN


OpenCL ™ (Open Computing Language) es un estándar abierto y libre de regalías para la programación paralela multiplataforma de diversos aceleradores que se encuentran en supercomputadoras, servidores en la nube, computadoras personales, dispositivos móviles y plataformas integradas. OpenCL mejora enormemente la velocidad y la capacidad de respuesta de un amplio espectro de aplicaciones en numerosas categorías de mercado, incluidas herramientas creativas profesionales, software científico y médico, procesamiento de la visión y entrenamiento e inferencia de redes neuronales.

Cómo se programan las aplicaciones OpenCL

OpenCL acelera las aplicaciones al descargar su código más intensivo en computación en procesadores o dispositivos aceleradores. Los desarrolladores de OpenCL utilizan lenguajes de kernel basados en C o C ++ para codificar programas que se pasan a través de un compilador de dispositivos para su ejecución en paralelo en dispositivos aceleradores.

 

Cómo se relaciona OpenCL con la familia de estándares de aceleración de Khronos

OpenCL proporciona a la industria la capa de ejecución ágil de procesadores 'cercana al metal' más baja para acelerar aplicaciones, bibliotecas y motores, y también proporciona un objetivo de generación de código para compiladores. A diferencia de las API 'solo para GPU', como Vulkan, OpenCL permite el uso de una amplia gama de aceleradores que incluyen CPU de múltiples núcleos, GPU, DSP, FPGA y hardware dedicado, como motores de inferencia.




Flexibilidad de implementación

Tipos de Volúmenes A medida que el panorama de la industria de plataformas y dispositivos se vuelve más complejo, las herramientas están evolucionando para permitir que las aplicaciones OpenCL se implementen en plataformas que no tienen controladores OpenCL nativos disponibles. Por ejemplo, el compilador clspv de código abierto y el traductor de API clvk permiten que las aplicaciones OpenCL se ejecuten en un tiempo de ejecución Vulkan. Esto brinda a los desarrolladores de OpenCL una flexibilidad significativa sobre dónde y cómo pueden implementar sus aplicaciones OpenCL.
